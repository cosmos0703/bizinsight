이번 학기 데이터 시각화 프로젝트로, "데이터를 통해 창업의 불확실성을 줄이자"는 목표로 시스템을 개발했습니다.

사용자가 선택한 업종에 맞춰 실시간으로 데이터를 필터링하여 시각화 컴포넌트로 전달하는 구조를 가지고 있습니다.

### 🔵 [1:00 ~ 1:40] 2. View 1: 사용자 맞춤형 설정 (UI/UX)
**(화면: Step 1 메인 화면. 마우스로 '외식업' 카테고리를 호버링)**

**[Na]**: 시스템은 총 4단계의 뷰(View)로 구성되어 있습니다.
첫 번째 뷰는 **데이터 필터링 인터페이스**입니다.
사용자가 복잡한 쿼리를 작성할 필요 없이, 직관적인 버튼과 슬라이더를 통해 원하는 데이터를 요청할 수 있습니다.

**(화면: '한식' 버튼 클릭, 예산 슬라이더 조절)**

**[Na]**: 시연을 위해 **'한식'** 업종을 선택하고, 창업 예산을 설정해보겠습니다.
이 인터페이스에서의 상호작용은 React의 State를 업데이트하고,
즉시 다음 뷰인 지도 컴포넌트의 렌더링 파라미터로 전달됩니다.

### 🔵 [1:40 ~ 3:00] 3. Main Scenario: '한식' 상권 분석 (Geospatial Viz)
**(화면: Step 2 지도 화면 진입. 서울시 전체에 히트맵이 펼쳐짐)**

**[Na]**: 두 번째 뷰, **지리적 시각화(Geospatial Visualization)**입니다.
**Google Maps API** 위에 **Recharts** 라이브러리를 연동하여 구현했습니다.

지금 지도에 나타난 붉은색 히트맵은 서울시 내 '한식' 업종의 **월 평균 매출** 분포입니다.
강남구와 종로구 일대가 붉게 표시되는 것을 볼 수 있는데,
이는 `getHeatmapColor` 함수가 실제 매출 데이터의 Min/Max 값을 계산하여 동적으로 색상을 매핑했기 때문입니다.

**(화면: 강남구 역삼동 부근으로 줌인. 마커 클릭 → 툴팁 팝업)**

**[Na]**: 특정 지역의 상세 데이터를 확인해보겠습니다.
마커를 클릭하면 툴팁이 나타나는데요,
이 정보는 정적인 텍스트가 아니라, 백엔드 데이터셋에서 해당 행정동의 **월 매출**, **유동인구**, **임대료** 정보를 실시간으로 쿼리하여 렌더링한 것입니다.

**(화면: 우측 하단 랭킹 리스트 스크롤)**

**[Na]**: 또한 우측 하단을 보시면, 단순 나열이 아닌 '추천 알고리즘'이 적용된 랭킹 리스트를 제공하여 사용자의 빠른 의사결정을 지원합니다.

### 🔵 [3:00 ~ 4:00] 4. Detailed Analysis & AI Insight
**(화면: Step 4 상세 분석 뷰. 그래프 애니메이션)**

**[Na]**: 이제 관심 지역을 선택하여 **상세 분석 뷰**로 이동합니다.
여기서는 **Recharts**의 `BarChart`와 `RadarChart`를 활용해 다차원 데이터를 시각화했습니다.

**(화면: '시간대별 유동인구' 바 차트와 레이더 차트 가리킴)**

**[Na]**: 바 차트를 통해 이 지역의 한식 업종이 점심시간인 12시에 매출 피크를 찍는다는 패턴을 발견할 수 있습니다.
데이터 시각화 수업에서 배운 것처럼, 단순히 수치를 보여주는 것을 넘어 '패턴'을 발견할 수 있도록 시각화했습니다.

**(화면: Gemini Analysis 섹션 (또는 텍스트가 타이핑되는 모습))**

**[Na]**: 그리고 여기 **AI 분석 리포트**가 있습니다.
이것은 현재 화면의 차트 데이터를 JSON 형태로 **Gemini API**에 전송하여 실시간으로 생성된 결과입니다.
데이터가 가리키는 패턴을 AI가 해석하여, "직장인 점심 타겟 마케팅이 필요하다"는 구체적인 전략을 텍스트로 풀어줍니다.

### 🟣 [4:00 ~ 4:40] 5. Secondary Scenario: '교습소' 데이터 전환
**(화면: 다시 Step 1으로 돌아가서 '교육/취미' -> '일반교습학원' 선택)**

**[Na]**: 이번에는 데이터셋을 변경했을 때 시스템이 어떻게 반응하는지 보여드리겠습니다.
만약 식당이 아니라 **'교습소'**를 창업한다면 어떨까요?
Step 1으로 돌아가 '일반교습학원'을 선택합니다.

**(화면: Step 2 지도로 이동. 강남구 대치동, 양천구 목동 등이 붉게 변함)**

**[Na]**: 지도의 히트맵 분포가 완전히 달라진 것을 확인할 수 있습니다.
아까와 달리, **대치동**과 **목동** 지역이 붉게 타오르고 있습니다.
이는 저희 시스템이 정해진 이미지를 보여주는 것이 아니라,
선택된 데이터셋의 특성을 실시간으로 반영하여 시각화를 재구성한다는 것을 증명합니다.

**(화면: 대치동 마커 클릭하여 10대 유동인구 비율 확인)**

**[Na]**: 상세 데이터에서도 '10대 유동인구' 비율이 압도적으로 높게 나타나는 것을 데이터를 통해 검증할 수 있습니다.
